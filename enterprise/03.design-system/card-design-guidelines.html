Card Design Guidelines

Design rules for illustrated cards used across the Powerhouse brand. These guidelines ensure visual consistency when creating new card layouts with background illustrations and gradient overlays.

Card Structure

Every illustrated card uses a three-layer system built with CSS pseudo-elements:

Layer 0 — Gradient overlay (::before): A radial gradient glow that sets the card's color mood. Positioned behind content but above the background.

Layer 0 — Illustration (::after): A large background illustration, partially cropped by the card boundaries. Sits at the same z-level as the gradient.

Layer 1 — Content (child elements): Text, icons, and buttons. Set position: relative; z-index: 1 on content wrappers to keep them above the pseudo-elements.

The card container requires:

  • position: relative — anchor point for pseudo-element positioning
  • overflow: hidden — crops illustrations at card edges
  • border-radius: 20px — standard card radius
  • background-color — solid dark surface color (e.g. #141719)
  • border: 1px solid rgba(255,255,255,0.06) — subtle surface border

Background Illustrations

Positioning Rules

Illustrations are tucked into the bottom-right corner of the card, with most of the image cropped off-screen. Only a portion of the illustration is visible — enough to add texture without competing with content.

Key principle: The illustration should feel like it's emerging from behind the bottom-right edge. Use large negative offsets in background-position to push the image beyond the card boundaries. The overflow: hidden on the card handles the cropping.

.card::after {
  content: '';
  position: absolute;
  inset: 0;
  pointer-events: none;
  z-index: 0;
  background-repeat: no-repeat;
  background-image: url('path/to/illustration.png');
  background-position: right -220px bottom -180px;
  background-size: 180%;
  opacity: 0.8;
}

Sizing and Offsets

The background-size controls how much of the illustration is visible. Larger sizes mean more cropping. Negative offsets in background-position push the image further off-screen.

Opacity

Default: 0.8 — Standard for pillar cards and hero cards. Illustrations are visible but don't overpower the gradient and content.

Product cards: 0.65 — Slightly more subdued to keep focus on the brand logos and CTAs.

Exception: 1.0 — Some illustrations with inherently low contrast (e.g. flowing/organic shapes) can use full opacity.

Gradient Overlays

Gradient Type

Always use radial-gradient with an ellipse shape. This creates an organic, focused glow rather than a flat wash of color.

Positioning

Pillar and hero cards: Gradient originates from the top-left corner. Use ellipse at 20% 0% or ellipse at 20% 20%.

Product cards: Gradient originates from the top-right area. Use ellipse at 70% -20%. This counterbalances the illustration in the bottom-right.

Implementation

.card::before {
  content: '';
  position: absolute;
  top: -30px; left: -30px; right: -30px;
  height: 280px;
  pointer-events: none;
  border-radius: 50%;
  filter: blur(30px);
  background: radial-gradient(ellipse at 20% 0%, rgba(R, G, B, 0.4–0.55), transparent 70%);
}

Content Layer

All text and interactive elements must sit above the illustration and gradient layers. Wrap content in a container with position: relative; z-index: 1.

For hero cards, limit content width to max-width: 55% to prevent text from overlapping the illustration on the right side.

Footer Wordmark

The gradient wordmark SVG is used as a large-scale background branding element at the very bottom of the page. It renders "POWERHOUSE" in outlined letters with a holographic gradient stroke.

Placement: Outside any padded container — directly on the body so it spans the full viewport width.

Container: Fixed height (~130px) with overflow: hidden. Image is positioned at top: 0 — the container height controls how much of the letters is visible.

Width: 110% with left: -5% — slightly oversized to push the first letter (P) off-screen, matching the reference.

Crop level: The bottom edge of the container should sit just above the horizontal crossbar of the H — only the tops of the letters are visible.

Dark backgrounds only: The SVG's gradient stroke (gray outlines with purple-to-blue iridescent accents) is designed for dark surfaces.

.footer-wordmark {
  position: relative;
  height: 130px;
  overflow: hidden;
}

.footer-wordmark img {
  position: absolute;
  top: 0;
  left: -5%;
  width: 110%;
  height: auto;
  pointer-events: none;
}

Checklist for New Cards

When creating a new illustrated card, verify each of these:

  • Card has position: relative; overflow: hidden
  • ::before used for radial gradient with filter: blur(30px)
  • ::after used for illustration with appropriate opacity (0.65–1.0)
  • Illustration is anchored to bottom-right with negative offsets
  • Gradient origin matches card type (top-left for pillars/hero, top-right for products)
  • Content wrapper has z-index: 1 to stay above pseudo-elements
  • Both pseudo-elements have pointer-events: none
  • Source illustration is 2048px+ for crisp rendering
