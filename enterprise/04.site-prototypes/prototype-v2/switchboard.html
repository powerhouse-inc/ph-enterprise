What Switchboard Does

Connect everything. Control everything.

Three audiences, one API layer.

For AI Agents

Every business operation becomes an MCP tool endpoint. AI agents interact with your data through typed, permission-scoped interfaces — no custom adapter per agent, no uncontrolled access.

For Existing Systems

GraphQL API connects your ERP, HRIS, document management, and financial systems. Each integration defines exactly what data flows in and out — nothing more.

For Applications

Fusion, Connect, and your custom applications all consume data through Switchboard. Authorization, scoping, and real-time sync are handled at this layer so individual apps don't need to re-implement them.

Integration Architecture

Standards-based. Auto-generated.
Scoped by default.

Standards-Based Connectivity

GraphQL for structured queries. MCP for AI agent tool access. Both auto-generated from your business schemas — when the schema evolves, the API surface updates automatically. No hand-written API code to maintain.

Data Scoping per Integration

Every connection defines a scope: which data types are accessible, which operations are permitted, what shape the data takes. No integration gets blanket access. Scoping is declarative, version-controlled, and auditable.

Multi-Layered Data Access

Three data access patterns: operational data (current state), historical data (complete operation history), and analytical data (read models built by event-driven processors).

Downstream Automation

Document change events flow through Switchboard to trigger agent workflows in Clint, dashboard updates in Fusion, webhook notifications, and automated follow-up operations.

Data Isolation Role

The privacy enforcement layer.

Tiered AI Model Routing

Data scoping controls which data reaches which AI model tier. Confidential data stays with local models. Only sanitized outputs reach commercial APIs.

Agent Permission Enforcement

Each agent's Renown identity determines which Switchboard endpoints it can access. Permissions are enforced at the API boundary — not by trusting the agent to behave.

Cross-System Boundary

Data from one system doesn't automatically flow to another. Every integration path is explicit, declarative, and auditable.

Audit at the Boundary

Every API call logged: caller identity, operation type, data accessed, timestamp. Your compliance team has a complete record of who accessed what and when.
